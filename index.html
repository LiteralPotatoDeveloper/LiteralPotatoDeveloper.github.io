Bootcamp 2021: Bitcoin Bubbles

A recreation of BitListen in Processing created in the Programming Bootcamp (2021) of City, University of London.
I tried to incorporate as many skills as I could to demonstrate my programming knowledge into this project, however there are many differences between C# and Java that I still need to learn.
These skills include classes, arrays, concurrency, web interactions, data visualization, sounds and more.
To do so, I used a WebSocket library to connect to the Blockchain API and requested to recieve messages for each unconfirmed transaction and block mined, which provided me with additional information such as the values of transactiosn involved, allowing me to display the data in the form of bubbles, along with playing sounds varying on the value.
Bubbles and Blocks are stored in an array to keep track and dispose of objects no longer visible and logarithmic algorithms are used to determine the speed of the bubble, based on value along with the volume and pitch of the sounds (Still experimental).
This project also involved minor modifications to a library in order to ensure compatibility, I edited the library to allow for larger text sizes to be recieved by my websocket.
This project has proven to be a nice introduction to data visualization and web communication along with helping me learn Java.
There are also many improvements that can be made, including bubble collision, optimization and even potential to become a game to click bubbles to collect bitcoin or view the transaction which can be implemented when I learn more Java

Check out the code on Github: https://github.com/LiteralPotatoDeveloper/BitcoinBubbles

Abyssal Tower

A game created for my 2022 Java Project in City, University Of London.

A quick summary of the game would be that it's a roguelike platformer with random levels.
The game currently consists of 3 random levels, which are picked at the end of each level, allowing the gameplay to go on infinitely.

The game features a functioning menu system, complete with functioning volume sliders by using the observer pattern to notify listeners.
There is also a pause menu ingame, allowing players to pause and save to continue at a different date.

The main challenge I had while creating this game was learning about java components that I needed, such as thread-safe javax swing timers, and constructing the gui.

Additionally, the implementation of animation was hard due to the lack of a BodyImage constructor which could take a raw image.

This would of allowed for more efficient usage of sprites, by internally splitting them This game could be improved by furthering my knowledge of OOP languages, allowing me to implement features more efficiently.
